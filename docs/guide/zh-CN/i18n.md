# 国际化（i18n）

EgoAdmin 采用前后端双层国际化架构，支持中文（zh-CN）和英文（en-US）两种语言。

## 概述

EgoAdmin 的国际化分为后端和前端两层。后端基于 `go-i18n` 库，通过嵌入式 TOML 文件管理消息模板，确保所有 API 错误提示均可按用户语言返回。前端基于 `vue-i18n`，管理页面标签、按钮文本、路由标题等 UI 文案。

两层通过 HTTP 请求头 `Accept-Language` 串联：前端根据当前语言设置发送请求头，Gateway 网关将其转化为 gRPC metadata 传递给下游服务，后端据此选择对应语言的消息模板。

默认语言为中文（zh-CN），所有功能均需同时维护中文和英文两套翻译文件。

::: tip 核心原则
不返回硬编码的用户可见字符串。所有面向用户的文本都必须通过 i18n 机制获取。
:::

::: warning 产品名称例外
产品名称和品牌标识保持不翻译，例如 "EgoAdmin" 在所有语言中保持不变。
:::

## 核心用法

### 后端：TOML 消息文件

后端消息定义在两个 TOML 文件中，采用嵌入式打包：

```text
internal/platform/i18n/
├── message.go                      # 核心 API
└── locales/
    ├── active.zh-CN.toml           # 中文消息
    └── active.en.toml              # 英文消息
```

添加新消息时，需要在两个文件中同时添加，使用 PascalCase 作为消息 ID：

```toml
# active.zh-CN.toml
[RoleInUseCount]
other = "该角色已被 {{.Count}} 个用户使用"

# active.en.toml
[RoleInUseCount]
other = "This role is used by {{.Count}} users"
```

### 后端：错误包装函数

平台提供三个错误包装函数，分别对应不同错误类型：

```go
import platformi18n "egoadmin/internal/platform/i18n"

// 业务失败（400）
return platformi18n.ErrorFailed(ctx, "RoleInUseCount", map[string]any{"Count": count})

// 未登录（401）
return platformi18n.ErrorNotLogin(ctx, "LoginExpired", nil)

// 无权限（403）
return platformi18n.ErrorAccessDenied(ctx, "DataPermissionOutOfScope", nil)
```

纯文本消息获取（不返回错误）：

```go
text := platformi18n.Message(ctx, "SomeMessageID")
text := platformi18n.Localize(ctx, "RoleInUseCount", map[string]any{"Count": count})
```

### 前端：vue-i18n 使用

前端通过 `t()` 函数获取翻译文案，支持嵌套键和插值：

```vue
<template>
  <el-button>{{ t('role.addSuccess') }}</el-button>
  <span>{{ t('common.total', { count: 10 }) }}</span>
</template>

<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
</script>
```

路由和菜单标题使用 i18n 键：

```ts
{
  path: '/role',
  meta: { title: 'menu.roleManagement' },
}
```

### 前端：语言切换组件

`LanguageSwitcher` 组件允许用户在运行时切换语言。切换后立即生效，无需刷新页面。Element Plus 组件库的全局 locale 也跟随切换，确保日期选择器、表格分页等内置组件的语言同步更新。

```vue
<!-- 简化示意 -->
<template>
  <el-dropdown @command="setLocale">
    <span>{{ currentLabel }}</span>
    <template #dropdown>
      <el-dropdown-menu>
        <el-dropdown-item
          v-for="opt in localeOptions"
          :key="opt.value"
          :command="opt.value"
        >
          {{ opt.label }}
        </el-dropdown-item>
      </el-dropdown-menu>
    </template>
  </el-dropdown>
</template>
```

## 配置示例

### 后端消息配置

TOML 消息格式遵循 go-i18n 规范。支持插值变量和复数形式：

```toml
# 简单消息
[LoginExpired]
other = "登录已过期"

# 带参数的消息
[RoleInUseCount]
other = "该角色已被 {{.Count}} 个用户使用"

# 复数形式（go-i18n 原生支持）
[ItemCount]
one = "{{.Count}} 个项目"
other = "{{.Count}} 个项目"
```

### 前端语言包配置

前端语言包位于 `web/src/i18n/package/` 目录，采用嵌套对象结构：

```ts
// web/src/i18n/package/zh-CN.ts
export default {
  common: {
    save: '保存',
    cancel: '取消',
    confirm: '确认',
    total: '共 {count} 条',
  },
  role: {
    addSuccess: '角色添加成功',
    deleteConfirm: '确定删除该角色吗？',
  },
  menu: {
    roleManagement: '角色管理',
    userManagement: '用户管理',
  },
  error: {
    loginExpired: '登录已过期，请重新登录',
  },
}
```

```ts
// web/src/i18n/package/en-US.ts
export default {
  common: {
    save: 'Save',
    cancel: 'Cancel',
    confirm: 'Confirm',
    total: '{count} items',
  },
  role: {
    addSuccess: 'Role added successfully',
    deleteConfirm: 'Are you sure to delete this role?',
  },
  menu: {
    roleManagement: 'Role Management',
    userManagement: 'User Management',
  },
  error: {
    loginExpired: 'Login expired, please log in again',
  },
}
```

### 前端 i18n 初始化

`web/src/i18n/index.ts` 负责初始化 vue-i18n 实例：

| 配置项 | 值 | 说明 |
|---|---|---|
| `legacy` | `false` | 使用 Composition API 模式 |
| `globalInjection` | `true` | 全局注入 `$t` |
| `fallbackLocale` | `'zh-CN'` | 回退语言 |
| `storageKey` | `'egoadmin-locale'` | localStorage 持久化键 |

语言选项配置：

| Locale 值 | 显示名 | Accept-Language |
|---|---|---|
| `zh-CN` | 简体中文 | `zh-CN` |
| `en` | English | `en-US` |

## 实战示例

### 完整流程：添加一个带国际化的新 API

以下以"删除角色"功能为例，展示从后端到前端的完整 i18n 集成。

**第一步：添加后端消息 ID**

在两个 TOML 文件中同时添加消息：

```toml
# internal/platform/i18n/locales/active.zh-CN.toml
[RoleDeleteSuccess]
other = "角色删除成功"

[RoleInUseCount]
other = "该角色已被 {{.Count}} 个用户使用，无法删除"

[RoleNotFound]
other = "角色不存在"
```

```toml
# internal/platform/i18n/locales/active.en.toml
[RoleDeleteSuccess]
other = "Role deleted successfully"

[RoleInUseCount]
other = "This role is used by {{.Count}} users and cannot be deleted"

[RoleNotFound]
other = "Role not found"
```

**第二步：在 Service 层使用错误包装**

```go
func (s *RoleService) DeleteRole(ctx context.Context, id uint64) error {
    role, err := s.repo.FindByID(ctx, id)
    if err != nil {
        return platformi18n.ErrorFailed(ctx, "RoleNotFound", nil)
    }

    count, err := s.repo.CountUsersByRoleID(ctx, id)
    if err != nil {
        return err
    }
    if count > 0 {
        return platformi18n.ErrorFailed(ctx, "RoleInUseCount",
            map[string]any{"Count": count})
    }

    if err := s.repo.Delete(ctx, id); err != nil {
        return err
    }
    return nil
}
```

**第三步：添加前端语言包**

```ts
// web/src/i18n/package/zh-CN.ts — 添加到 role 命名空间
role: {
  addSuccess: '角色添加成功',
  deleteSuccess: '角色删除成功',
  deleteConfirm: '确定删除该角色吗？',
  inUseCannotDelete: '该角色正在使用中，无法删除',
}

// web/src/i18n/package/en-US.ts — 对应添加
role: {
  addSuccess: 'Role added successfully',
  deleteSuccess: 'Role deleted successfully',
  deleteConfirm: 'Are you sure to delete this role?',
  inUseCannotDelete: 'This role is in use and cannot be deleted',
}
```

**第四步：在组件中使用**

```vue
<template>
  <el-button @click="handleDelete">{{ t('common.delete') }}</el-button>
</template>

<script setup lang="ts">
import { useI18n } from 'vue-i18n'
import { ElMessageBox, ElMessage } from 'element-plus'

const { t } = useI18n()

async function handleDelete() {
  await ElMessageBox.confirm(
    t('role.deleteConfirm'),
    t('common.confirm')
  )
  try {
    await deleteRoleApi(id)
    ElMessage.success(t('role.deleteSuccess'))
  } catch (e) {
    // API 错误消息已由后端 i18n 本地化，前端直接展示
    ElMessage.error(e.message)
  }
}
</script>
```

::: tip 错误消息展示
后端返回的错误消息已根据用户语言本地化，前端无需二次翻译，直接展示 `error.message` 即可。
:::

## 工作原理

### 语言传递链路

```
前端 (vue-i18n)
  ↓ 设置 Accept-Language 请求头
HTTP 请求 → Gateway 网关
  ↓ 转化为 grpcgateway-accept-language metadata
gRPC 请求 → User 服务
  ↓ 从 metadata 读取 accept-language
后端 (go-i18n) → 选择对应语言 localizer → 返回本地化消息
```

前端通过 `getAcceptLanguage()` 函数获取当前语言对应的 Accept-Language 值，该值在 HTTP 请求中发送。Gateway 将其转化为 gRPC metadata（`accept-language` 或 `grpcgateway-accept-language`），下游服务通过 `localizer(ctx)` 函数从 context 中读取语言偏好。

### 后端 localizer 解析逻辑

`platformi18n.localizer(ctx)` 函数从 context 中提取 Accept-Language，创建对应语言的 `i18n.Localizer`。如果未找到或为空，回退到默认语言 `zh-CN`。

```text
// 简化逻辑示意
func localizer(ctx context.Context) *i18n.Localizer {
    lang := extractAcceptLanguage(ctx) // 从 gRPC metadata 读取
    if lang == "" {
        lang = defaultLanguage
    }
    return i18n.NewLocalizer(bundle, lang)
}
```

消息查找流程：

1. 从 context 中的 gRPC metadata 读取 `accept-language` 键
2. 若为空，尝试读取 `grpcgateway-accept-language` 键
3. 若仍为空，使用默认语言 `zh-CN`
4. 创建 `i18n.Localizer` 并调用 `Localize()` 获取消息
5. 若消息 ID 不存在，返回原始 message ID 作为兜底

### 前端语言持久化

前端语言偏好通过 `localStorage` 持久化（键名 `egoadmin-locale`）。首次加载时按以下优先级确定语言：

1. `localStorage` 中已保存的语言
2. 浏览器 `navigator.language`
3. 回退到 `zh-CN`

切换语言时，通过 `setLocale()` 函数同时更新 vue-i18n locale、localStorage，并派发 `egoadmin-locale-changed` 自定义事件通知其他组件。

### Gateway 错误保持机制

Gateway 在转发下游服务的错误时，需要将 gRPC status details 反序列化回 `*eerrors.EgoError`，保持原始错误消息不变。这样下游服务通过 `platformi18n.ErrorFailed()` 设置的本地化消息能完整传递给前端。

## 常见问题

### 后端返回的消息是 message ID 而非翻译文本

**原因：** TOML 文件中缺少对应的消息 ID，或消息 ID 拼写不一致。

**解决：** 检查 `active.zh-CN.toml` 和 `active.en.toml` 两个文件是否都包含该消息 ID。`localizer` 在找不到消息时会原样返回 message ID。

```go
// message.go 中的回退逻辑
msg, err := localizer(ctx).Localize(&i18n.LocalizeConfig{
    MessageID:    messageID,
    TemplateData: data,
})
if err != nil {
    return messageID // 找不到时返回原始 ID
}
```

### 用户切换语言后 API 错误仍是旧语言

**原因：** Gateway 未正确转发 `Accept-Language` 到下游 gRPC 服务。

**解决：** 确认 Gateway 的 HTTP 转 gRPC 中间件是否保留了 `accept-language` metadata。检查前端请求头是否包含正确的 Accept-Language 值。

### 前端页面部分文案未翻译

**原因：** 使用了硬编码字符串而非 `t()` 函数，或语言包中缺少对应键。

**解决：** 搜索代码中的中文/英文硬编码字符串，替换为 `t('key')` 调用，并确保两个语言包都添加了对应键。排查步骤：

```bash
# 搜索可能遗漏的硬编码中文
grep -rn '[一-鿿]' web/src/ --include='*.vue' --include='*.ts' | grep -v 'node_modules' | grep -v 'package/'
```

### 插值变量不生效

**原因：** 后端 TOML 使用 <code v-pre>{{.VarName}}</code>（Go 模板语法），前端使用 `{varName}`（vue-i18n 语法），两者格式不同。

**解决：** 后端模板使用 <code v-pre>{{.Count}}</code>，前端语言包使用 `{count}`。注意不要混用。

### 新增 TOML 消息后编译报错

**原因：** TOML 文件使用 `//go:embed` 嵌入，修改后需要重新编译。

**解决：** 直接 `go build` 或 `make build` 即可，embed 文件在编译时自动包含。无需额外的代码生成步骤。
