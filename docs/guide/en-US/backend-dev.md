# Backend Development Guide

For developers who need to add or modify backend APIs in EgoAdmin. This page covers the full chain from Proto definition to the persistence layer, helping you get started quickly with day-to-day development under the DDD layered architecture.

## Overview

EgoAdmin backend uses a Domain-Driven Design (DDD) layered architecture. Service entrypoints live in `cmd/<service>`, and business code resides in `internal/app/<service>`. Each service is organized into five layers: server, controller, application, domain, and adapter, with a strict unidirectional dependency flow: server -> controller -> application -> domain. The adapter layer implements interfaces defined in the domain layer.

The core development flow is: **Define Proto -> make gen to generate code -> Implement Controller -> Write Application/Domain logic -> Implement persistence -> Update permissions and frontend -> Test and verify**. Once you understand this chain, adding a new API is simply a matter of filling in code at each layer.

This guide uses the user service's **Role** module as a running example, showing real code patterns from each layer.

## Core Usage

### 1. Define the Proto Contract

All APIs start from `api/proto/<service>/v1/*.proto`. The Proto defines RPC methods, request/response messages, validation tags, and copier mapping tags.

```protobuf
// api/proto/user/v1/role.proto
syntax = "proto3";
package user.v1;

import "google/api/annotations.proto";
import "google/api/field_behavior.proto";
import "tagger/tagger.proto";
import "protoc-gen-openapiv2/options/annotations.proto";

// RoleService manages roles
service RoleService {
  // AddRole creates a new role
  rpc AddRole(AddRoleRequest) returns (AddRoleResponse) {
    option (google.api.http) = {
      post: "/user.v1.RoleService/AddRole",
      body: "*"
    };
  };

  // GetRoleList retrieves a paginated role list
  rpc GetRoleList(GetRoleListRequest) returns (GetRoleListResponse) {
    option (google.api.http) = {
      post: "/user.v1.RoleService/GetRoleList",
      body: "*"
    };
  };
}

message AddRoleRequest {
  Role role = 1 [
    (tagger.tags) = "validate:\"required\" label:\"Role Info\"",
    (google.api.field_behavior) = REQUIRED
  ];
}

message AddRoleResponse {
  uint64 id = 1;
}

message GetRoleListRequest {
  int32 page = 1 [
    (tagger.tags) = "validate:\"required,gte=1\" label:\"Page\"",
    (google.api.field_behavior) = REQUIRED
  ];
  int32 limit = 2 [
    (tagger.tags) = "validate:\"required,gte=1,lte=50\" label:\"Page Size\"",
    (google.api.field_behavior) = REQUIRED
  ];
  string sort = 3 [(tagger.tags) = "validate:\"omitempty,gte=1\" label:\"Sort Field\""];
  string order = 4 [(tagger.tags) = "validate:\"omitempty,oneof=desc asc\" label:\"Sort Order\""];
  string name = 5 [(tagger.tags) = "validate:\"omitempty,gte=1\" label:\"Role Name\""];
}

message GetRoleListResponse {
  int32 total = 1;
  repeated Role roles = 2;
}

message Role {
  uint64 id = 1 [(tagger.tags) = "copier:\"ID\""];
  string name = 5 [
    (tagger.tags) = "validate:\"required\" label:\"Role Name\"",
    (google.api.field_behavior) = REQUIRED
  ];
  int32 typ = 6 [
    (tagger.tags) = "validate:\"required,oneof=1\" label:\"Role Type\"",
    (google.api.field_behavior) = REQUIRED
  ];
  int32 data_perm = 7 [
    (tagger.tags) = "validate:\"required,oneof=1 2 3 4\" label:\"Data Permission\"",
    (google.api.field_behavior) = REQUIRED
  ];
  string desc = 8 [(tagger.tags) = "validate:\"omitempty\" label:\"Description\""];
  string view_menus = 9 [(tagger.tags) = "validate:\"omitempty\" label:\"Menu Permissions\""];
  string uses = 10 [(tagger.tags) = "validate:\"omitempty\" label:\"Feature Permissions\""];
}
```

**Key points:**
- `validate` tags in `(tagger.tags)` are injected into Go structs during code generation and enforced automatically by middleware.
- `copier:"ID"` tags are used by `jinzhu/copier` for field mapping between protobuf messages and store models.
- HTTP routes use POST + gRPC path style (e.g., `/user.v1.RoleService/AddRole`).

### 2. Run Code Generation

```bash
# Generate proto, Go code, and Wire code
make gen
```

Generated outputs include:
- `api/gen/go/user/v1/*.pb.go` -- protobuf messages and gRPC service interfaces
- `api/gen/go/user/v1/*.pb.validate.go` -- validation logic
- `api/gen/go/user/v1/*.pb.gw.go` -- protoc-gen-go-http HTTP compatibility layer

::: warning
Never edit generated files manually. You must re-run `make gen` after every proto change.
:::

### 3. Implement the Controller

The Controller layer implements the gRPC service interfaces generated by `make gen`. Its responsibilities are: receive protobuf requests, convert to store/service inputs, call business logic, and map results back to protobuf responses.

```go
// internal/app/user/controller/role_grpc.go
package controller

import (
    "context"
    "errors"
    "math"

    userv1 "github.com/egoadmin/egoadmin/api/gen/go/user/v1"
    roledomain "github.com/egoadmin/egoadmin/internal/app/user/domain/role"
    store "github.com/egoadmin/egoadmin/internal/app/user/internal/store"
    "github.com/egoadmin/egoadmin/internal/app/user/service"
    platformi18n "github.com/egoadmin/egoadmin/internal/platform/i18n"
    "github.com/egoadmin/elib/pkg/util/xorm"
    "github.com/jinzhu/copier"
)

// RoleGRPC is the role gRPC controller
type RoleGRPC struct {
    role *service.RoleService
}

func NewRoleGRPCController(role *service.RoleService) *RoleGRPC {
    return &RoleGRPC{role: role}
}

// AddRole creates a new role
func (s *RoleGRPC) AddRole(ctx context.Context, in *userv1.AddRoleRequest) (
    out *userv1.AddRoleResponse, err error,
) {
    // 1) Initialize out to guarantee a non-nil return value
    out = &userv1.AddRoleResponse{}

    // 2) Convert protobuf request to store model
    role := &store.RoleModel{
        Name:      in.GetRole().GetName(),
        Typ:       in.GetRole().GetTyp(),
        DataPerm:  in.GetRole().GetDataPerm(),
        Uses:      in.GetRole().GetUses(),
        ViewMenus: in.GetRole().GetViewMenus(),
        Desc:      in.GetRole().GetDesc(),
    }

    // 3) Call the service layer
    err = s.role.AddRole(ctx, role)
    if err = mapRoleError(ctx, err); err != nil {
        return
    }

    // 4) Write the result back to the response
    out.Id = role.ID
    return
}

// GetRoleList retrieves a paginated role list
func (s *RoleGRPC) GetRoleList(ctx context.Context, in *userv1.GetRoleListRequest) (
    out *userv1.GetRoleListResponse, err error,
) {
    out = &userv1.GetRoleListResponse{}

    // Build pagination options from the protobuf request
    pgopt := xorm.PaginateOption{
        Page:  int(in.GetPage()),
        Limit: int(in.GetLimit()),
        Sort:  in.GetSort(),
        Order: in.GetOrder(),
    }

    // Call service to get the list
    roles, total, err := s.role.GetRoleList(ctx, in.GetName(), pgopt)
    if err != nil {
        return
    }
    if total > math.MaxInt32 {
        err = platformi18n.ErrorFailed(ctx, "RoleCountExceeded", nil)
        return
    }

    out.Total = int32(total)
    out.Roles = make([]*userv1.Role, 0, len(roles))

    // Use copier to batch-map store models to protobuf messages
    if err = copier.Copy(&out.Roles, &roles); err != nil {
        return
    }
    return
}

// mapRoleError converts domain errors to internationalized errors
func mapRoleError(ctx context.Context, err error) error {
    var inUse roledomain.InUseError
    switch {
    case err == nil:
        return nil
    case errors.Is(err, roledomain.ErrNameExists):
        return platformi18n.ErrorFailed(ctx, "RoleNameExists", nil)
    case errors.As(err, &inUse):
        return platformi18n.ErrorFailed(ctx, "RoleInUseCount",
            map[string]any{"Count": inUse.Count})
    case errors.Is(err, roledomain.ErrInUse):
        return platformi18n.ErrorFailed(ctx, "RoleInUse", nil)
    default:
        return err
    }
}
```

**Controller rules:**
- The first thing in every method is `out = &XxxResponse{}` to initialize the return value.
- Use `in.GetXxx()` to access nested message fields (proto3 zero-value semantics safe).
- Validation is handled automatically by middleware (based on `validate` tags in proto), so no manual validation in controllers.
- Use `copier.Copy` for bulk mapping between protobuf and store models.
- Do not put direct GORM queries in the controller.

### 4. Write the Service / Application Layer

The Service layer (or Application layer in DDD) is responsible for transactions, locks, permission validation, cross-service orchestration, and cache invalidation.

```go
// internal/app/user/service/role_service.go
package service

import (
    "context"
    "errors"

    "github.com/egoadmin/egoadmin/internal/app/user/application"
    roledomain "github.com/egoadmin/egoadmin/internal/app/user/domain/role"
    store "github.com/egoadmin/egoadmin/internal/app/user/internal/store"
    platformi18n "github.com/egoadmin/egoadmin/internal/platform/i18n"
    "github.com/egoadmin/elib/pkg/util/xorm"
)

type RoleService struct {
    Options
}

func NewRoleService(options Options) *RoleService {
    return &RoleService{Options: options}
}

// AddRole creates a new role
func (s *RoleService) AddRole(ctx context.Context, role *store.RoleModel) error {
    // Data permission validation
    scope, err := s.DataScope(ctx)
    if err != nil {
        return err
    }
    if err = scope.EnforceAssignableDataPerm(ctx, role.DataPerm); err != nil {
        return err
    }
    if !scope.IsAdmin {
        role.OwnerUserID = scope.UserID
        role.OwnerDeptID = scope.DeptID
    }

    // Call application use case (domain orchestration)
    result, err := s.RoleUseCase.CreateRole(ctx, application.SaveRoleCommand{
        Name:        role.Name,
        Type:        role.Typ,
        DataPerm:    role.DataPerm,
        OwnerUserID: role.OwnerUserID,
        OwnerDeptID: role.OwnerDeptID,
        Uses:        role.Uses,
        ViewMenus:   role.ViewMenus,
        Desc:        role.Desc,
    })
    if err != nil {
        return mapRoleDomainError(ctx, err)
    }
    role.ID = result.ID
    return nil
}

// GetRoleList retrieves a paginated role list
func (s *RoleService) GetRoleList(ctx context.Context, name string, pgopt xorm.PaginateOption) (
    roles []*store.RoleModel, total int64, err error,
) {
    scope, err := s.DataScope(ctx)
    if err != nil {
        return nil, 0, err
    }
    // Data permission scope is injected as a GORM scope into the query
    roles, total, err = s.Role.GetList(ctx, name, pgopt, scope.RoleScope())
    return
}
```

**Service rules:**
- Transactions are controlled at the service layer (use `mysql.Transaction` when spanning multiple store operations).
- Duplicate checks, delete guards, status transitions, and other business rules are implemented here or in the domain layer.
- Returns store models or service DTOs, not protobuf messages.
- Cross-service calls go through `internal/client/*` packages.

### 5. Define the Domain Layer

The Domain layer defines aggregate roots, value objects, domain errors, and Repository interfaces. It has no dependency on any infrastructure package.

```go
// internal/app/user/domain/role/role.go
package role

type Role struct {
    ID          uint64
    Name        string
    Type        int32
    BuiltIn     int32
    DataPerm    int32
    OwnerUserID uint64
    OwnerDeptID uint64
    Uses        string
    ViewMenus   string
    Desc        string
    Policies    []PermissionPolicy
}

type PermissionPolicy struct {
    Service string
    Method  string
}

// NormalizePolicies deduplicates and cleans permission policies
func NormalizePolicies(policies []PermissionPolicy) []PermissionPolicy {
    normalized := make([]PermissionPolicy, 0, len(policies))
    seen := make(map[string]struct{}, len(policies))
    for _, p := range policies {
        service := strings.ToUpper(strings.TrimSpace(p.Service))
        method := strings.ToUpper(strings.TrimSpace(p.Method))
        if service == "" || method == "" {
            continue
        }
        key := service + "/" + method
        if _, ok := seen[key]; ok {
            continue
        }
        seen[key] = struct{}{}
        normalized = append(normalized, PermissionPolicy{
            Service: service, Method: method,
        })
    }
    return normalized
}
```

### 6. Implement the Persistence Layer (Store / Adapter)

There are two coexisting persistence paths:

- **Migration-era path** (`internal/app/<service>/internal/store`): Contains GORM models, Repository interface implementations, Scopes, and transaction-aware helpers.
- **DDD target path** (`internal/app/<service>/adapter/persistence/mysql`): Implements Repository interfaces defined in the domain layer, with domain model <-> persistence model conversion.

#### Migration-Era Store Example

```go
// internal/app/user/internal/store/role.go
package store

import (
    "context"
    "github.com/egoadmin/egoadmin/internal/platform/database/mysql"
    "github.com/gotomicro/ego-component/egorm"
    "gorm.io/gorm"
)

// RoleModel is the role GORM model
type RoleModel struct {
    xorm.Model
    Name        string `gorm:"index;type:varchar(255);not null;default:'';comment:Role name"`
    Typ         int32  `gorm:"int(10);not null;default:1;comment:Type"`
    BuiltIn     int32  `gorm:"int(10);not null;default:2;comment:1=built-in,2=normal"`
    DataPerm    int32  `gorm:"int(10);not null;default:1;comment:Data permission"`
    OwnerUserID uint64 `gorm:"index;type:bigint(20) unsigned;not null;default:0"`
    OwnerDeptID uint64 `gorm:"index;type:bigint(20) unsigned;not null;default:0"`
    Uses        string `gorm:"type:text;not null;comment:Feature permissions"`
    ViewMenus   string `gorm:"type:text;not null;comment:Menu permissions"`
    Desc        string `gorm:"type:varchar(255);not null;default:'';comment:Description"`
    Policies    []RolePermissionPolicy `gorm:"foreignKey:RoleModelID;references:ID"`
}

func (RoleModel) TableName() string { return "role" }

func (m *RoleModel) SetID(id uint64) {
    if m.ID == 0 { m.ID = id }
}

func (m *RoleModel) BeforeCreate(tx *gorm.DB) error {
    return mysql.SetID(m)
}

// Role is the role Repository implementation
type Role struct {
    cc *egorm.Component
}

func NewRole(db *egorm.Component) RoleInterface {
    return &Role{cc: db}
}

// Add creates a role and its permission policies
func (m *Role) Add(ctx context.Context, role *RoleModel) error {
    role.Policies = normalizePermissionPolicies(role.Policies)
    db := mysql.DBWithContext(ctx, m.cc)
    return db.Create(&role).Error
}

// Delete removes a role and its permission policies
func (m *Role) Delete(ctx context.Context, id uint64) error {
    return mysql.Transaction(ctx, m.cc, func(tx *gorm.DB) error {
        if err := tx.Where(map[string]any{"role_model_id": id}).Delete(&RolePermissionPolicy{}).Error; err != nil {
            return err
        }
        return tx.Unscoped().Delete(&RoleModel{Model: xorm.Model{ID: id}}).Error
    })
}

// GetList queries roles with pagination
func (m *Role) GetList(ctx context.Context, name string, opt xorm.PaginateOption,
    scopes ...func(*gorm.DB) *gorm.DB,
) ([]*RoleModel, int64, error) {
    db := mysql.DBWithContext(ctx, m.cc)
    scopes = append(scopes, roleScopeNameLike(name))
    if opt.Sort == "" {
        opt.Sort = "created_at"
        opt.Order = "desc"
    }

    var total int64
    if err := db.Scopes(scopes...).Model(&RoleModel{}).Count(&total).Error; err != nil {
        return nil, 0, err
    }
    scopes = append(scopes, xorm.WithScopePaginate(opt)...)

    var roles []*RoleModel
    err := db.Scopes(scopes...).Find(&roles).Error
    return roles, total, err
}
```

## Real-World Example: Adding a "Menu Management" API

Below is a walkthrough for adding a complete CRUD API from scratch.

### Step 1: Proto Definition

```protobuf
// api/proto/user/v1/menu.proto
syntax = "proto3";
package user.v1;

import "google/api/annotations.proto";
import "google/api/field_behavior.proto";
import "tagger/tagger.proto";

service MenuService {
  rpc AddMenu(AddMenuRequest) returns (AddMenuResponse) {
    option (google.api.http) = { post: "/user.v1.MenuService/AddMenu", body: "*" };
  };
  rpc GetMenuList(GetMenuListRequest) returns (GetMenuListResponse) {
    option (google.api.http) = { post: "/user.v1.MenuService/GetMenuList", body: "*" };
  };
}

message AddMenuRequest {
  string name = 1 [
    (tagger.tags) = "validate:\"required\" label:\"Menu Name\"",
    (google.api.field_behavior) = REQUIRED
  ];
  uint64 parent_id = 2;
  int32 sort = 3;
  string path = 4;
}

message AddMenuResponse {
  uint64 id = 1;
}

message GetMenuListRequest {}

message GetMenuListResponse {
  repeated Menu menus = 1;
}

message Menu {
  uint64 id = 1 [(tagger.tags) = "copier:\"ID\""];
  string name = 2;
  uint64 parent_id = 3;
  int32 sort = 4;
  string path = 5;
}
```

Run `make gen`.

### Step 2: Store Layer

```go
// internal/app/user/internal/store/menu.go
package store

import (
    "context"
    "github.com/egoadmin/egoadmin/internal/platform/database/mysql"
    "github.com/gotomicro/ego-component/egorm"
    "gorm.io/gorm"
)

type MenuModel struct {
    xorm.Model
    Name     string `gorm:"type:varchar(255);not null;comment:Menu name"`
    ParentID uint64 `gorm:"index;type:bigint(20) unsigned;not null;default:0;comment:Parent menu ID"`
    Sort     int32  `gorm:"int(10);not null;default:0;comment:Sort order"`
    Path     string `gorm:"type:varchar(255);not null;default:'';comment:Route path"`
}

func (MenuModel) TableName() string { return "menu" }
func (m *MenuModel) SetID(id uint64) { if m.ID == 0 { m.ID = id } }
func (m *MenuModel) BeforeCreate(tx *gorm.DB) error { return mysql.SetID(m) }

type Menu struct {
    cc *egorm.Component
}

func NewMenu(db *egorm.Component) MenuInterface {
    return &Menu{cc: db}
}

func (m *Menu) Add(ctx context.Context, menu *MenuModel) error {
    db := mysql.DBWithContext(ctx, m.cc)
    return db.Create(menu).Error
}

func (m *Menu) GetAll(ctx context.Context) ([]*MenuModel, error) {
    db := mysql.DBWithContext(ctx, m.cc)
    var menus []*MenuModel
    err := db.Order("sort ASC, id ASC").Find(&menus).Error
    return menus, err
}
```

### Step 3: Controller

```go
// internal/app/user/controller/menu_grpc.go
package controller

import (
    "context"
    userv1 "github.com/egoadmin/egoadmin/api/gen/go/user/v1"
    store "github.com/egoadmin/egoadmin/internal/app/user/internal/store"
    "github.com/egoadmin/egoadmin/internal/app/user/service"
    "github.com/jinzhu/copier"
)

type MenuGRPC struct {
    menu *service.MenuService
}

func NewMenuGRPCController(menu *service.MenuService) *MenuGRPC {
    return &MenuGRPC{menu: menu}
}

func (s *MenuGRPC) AddMenu(ctx context.Context, in *userv1.AddMenuRequest) (
    out *userv1.AddMenuResponse, err error,
) {
    out = &userv1.AddMenuResponse{}
    menu := &store.MenuModel{
        Name:     in.GetName(),
        ParentID: in.GetParentId(),
        Sort:     in.GetSort(),
        Path:     in.GetPath(),
    }
    if err = s.menu.AddMenu(ctx, menu); err != nil {
        return
    }
    out.Id = menu.ID
    return
}

func (s *MenuGRPC) GetMenuList(ctx context.Context, in *userv1.GetMenuListRequest) (
    out *userv1.GetMenuListResponse, err error,
) {
    out = &userv1.GetMenuListResponse{Menus: make([]*userv1.Menu, 0)}
    menus, err := s.menu.GetAll(ctx)
    if err != nil {
        return
    }
    if err = copier.Copy(&out.Menus, &menus); err != nil {
        return
    }
    return
}
```

## How It Works

### Wire Dependency Injection

EgoAdmin uses google/wire for compile-time dependency injection. Each service's `server/wire.go` defines the complete dependency graph:

```go
//go:build wireinject
// +build wireinject

package server

import (
    usermysql "github.com/egoadmin/egoadmin/internal/app/user/adapter/persistence/mysql"
    "github.com/egoadmin/egoadmin/internal/app/user/application"
    "github.com/egoadmin/egoadmin/internal/app/user/controller"
    "github.com/egoadmin/egoadmin/internal/app/user/service"
    "github.com/google/wire"
)

func NewApp() (*App, error) {
    panic(wire.Build(
        // Infrastructure
        newEgo,
        newEgoReady,
        newConfig,
        newHealth,
        newCasbin,
        newValidate,
        wire.Struct(new(Options), "*"),

        // Layer ProviderSets
        application.ProviderSet,
        usermysql.ProviderSet,
        controller.ProviderSet,
        service.ProviderSet,

        // Interface bindings
        wire.Bind(new(application.RoleAssignments), new(*usermysql.UserRepository)),

        // Final assembly
        newApp,
    ))
}
```

Running `make gen` generates `wire_gen.go`, resolving all dependencies at compile time. There is zero reflection overhead at runtime.

### Layer Responsibilities Overview

```text
cmd/<service>/main.go          Entrypoint, starts the server
internal/app/<service>/
  server/                       EGO app, logging, config, registry, health, gRPC/HTTP, migrations
  controller/                   Implements generated gRPC server interfaces
  application/                  Transactions, locks, permission validation, cross-service orchestration, DTM
  domain/<aggregate>/           Aggregates, entities, value objects, domain errors, Repository interfaces
  adapter/persistence/mysql/    GORM models, Repository implementations, Scopes, migration model lists
  service/                      Business service layer (compat period, wraps store + use case)
  internal/store/               GORM models, Repository interfaces/implementations, Scopes
internal/platform/              Shared infrastructure (database, cache, config, health checks, etc.)
internal/component/             Reusable EGO-style components (authsession, idgen, etc.)
internal/client/                Service-to-service gRPC call wrappers
```

**Dependency direction:**
```text
server -> controller -> application -> domain
adapter -> domain + platform
store  -> platform
```

::: warning
Cross-service data access must go through `internal/client/*` packages. Never import another service's store or domain packages directly.
:::

## Common Issues

### Q: make gen fails with "protoc not found"

**Cause:** protoc is not installed or not in PATH.

**Solution:**
```bash
# macOS
brew install protobuf

# Linux
apt-get install -y protobuf-compiler

# Verify version
protoc --version
```

### Q: copier.Copy maps fields as zero values

**Cause:** Proto-generated Go fields and store model fields have different names. copier matches by field name.

**Solution:** Add a `copier` tag to the proto message field:
```protobuf
uint64 id = 1 [(tagger.tags) = "copier:\"ID\""];
```
Or add a `copier` struct tag to the Go model field.

### Q: GORM queries missing data permission filtering

**Cause:** The service layer did not pass `DataScope` as a GORM scope to the store query method.

**Solution:** Ensure the service layer obtains `DataScope` and passes its `RoleScope()` / `UserScope()` methods (which return `func(*gorm.DB) *gorm.DB`) to the store method:
```go
scope, err := s.DataScope(ctx)
roles, total, err := s.Role.GetList(ctx, name, pgopt, scope.RoleScope())
```

### Q: Wire code generation reports circular dependency

**Cause:** Circular references between ProviderSets, typically A imports B and B imports A.

**Solution:** Check that the dependency direction is correct. The domain layer should not import any infrastructure packages. Cross-service calls should use `internal/client/*`. If decoupling is needed, define an interface and bind it via `wire.Bind`.

### Q: Controller returns "permission denied"

**Cause:** After registering an API with the gateway, the gateway validates the current user's API permissions. If the permission list is not updated, requests will be blocked.

**Solution:**
1. Verify the service name and method name casing in the proto is correct.
2. Check the new API in the admin panel's role permission settings.
3. Confirm the gateway's API dictionary has synced the latest proto definitions (restart gateway after `make gen`).

### Q: Should I use migration-era store or DDD adapter?

**Cause:** Both persistence paths coexist in the project.

**Solution:**
- New modules should use `adapter/persistence/mysql` (DDD path), defining Repository interfaces in the domain layer.
- Existing modules still in `internal/store` should continue iterating in that path; do not mix them.
- The long-term goal is for all modules to migrate to `adapter/persistence/mysql`.
