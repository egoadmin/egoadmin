# Component System

Reusable capabilities live under `internal/component` and shared infrastructure lives under `internal/platform`.

## Main Components

| Component | Path | Responsibility |
|-----------|------|----------------|
| AuthSession | `internal/component/authsession` | Bearer auth, JWT session, multi-login, heartbeat offline |
| LoginCrypto | `internal/component/logincrypto` | RSA-OAEP challenge-based password transport |
| IDGen | `internal/component/idgen` | Snowflake IDs, segments, leases, ID codec |
| Redis | `internal/component/eredis` | Redis wrapper and utilities |
| ETUSUpload | `internal/component/etusupload` | TUS resumable upload |

### Directory Layout


## Component Lifecycle

Components implement a unified lifecycle interface registered with the shutdown manager:

```go
// internal/platform/shutdown/manager.go
type Closer interface {
  Close() error
}
```

Typical lifecycle:

```text
Server starts
  -> Wire dependency injection, construct components
  -> Component initialization (connection pools, connections, cache warmup)
  -> Register with shutdown.Manager
  -> Service begins accepting requests

Server receives SIGTERM/SIGINT
  -> shutdown.Manager executes Close in LIFO order
  -> Components release resources (close connections, flush buffers)
  -> Process exits
```

Component Close example:

```go
// internal/component/eredis/eredis.go
type ERedis struct {
  client redis.UniversalClient
}

func (r *ERedis) Close() error {
  return r.client.Close()
}
```

::: tip
Component `Close()` methods are automatically called by `shutdown.Manager` on process exit. Business code does not need to manually close components.
:::

## Wire Provider Set Pattern

Each component exposes its constructor via a Wire provider set. Services reference these sets in their Wire injectors:

```go
// internal/component/authsession/wire.go
package authsession

import "github.com/google/wire"

var ProviderSet = wire.NewSet(
  NewAuthSession,
  NewAuthMiddleware,
)

// internal/component/eredis/wire.go
package eredis

import "github.com/google/wire"

var ProviderSet = wire.NewSet(
  NewERedis,
)

// internal/component/idgen/wire.go
package idgen

import "github.com/google/wire"

var ProviderSet = wire.NewSet(
  NewIDGenerator,
  NewIDCodec,
)
```

Service Wire injector:

```go
// internal/app/user/wire.go
//go:build wireinject

package user

import (
  "github.com/google/wire"
  "github.com/egoadmin/egoadmin/internal/component/authsession"
  "github.com/egoadmin/egoadmin/internal/component/eredis"
  "github.com/egoadmin/egoadmin/internal/component/idgen"
  "github.com/egoadmin/egoadmin/internal/component/logincrypto"
)

func InitializeServer() (*Server, error) {
  wire.Build(
    authsession.ProviderSet,
    eredis.ProviderSet,
    idgen.ProviderSet,
    logincrypto.ProviderSet,
    // ... business bindings
    NewServer,
  )
  return nil, nil
}
```

::: warning
Do not edit `wire_gen.go`. It is auto-generated by `make gen`.
:::

```toml
heartbeatOfflineSeconds = 660

```

### AuthContext Injection

The AuthSession middleware injects an AuthContext into every request:

```go
type AuthContext struct {
  UserID    uint64
  Username  string
  IsAdmin   bool
  IsRoot    bool
  DataScope DataScopeType
  DeptID    uint64
  DeptIDs   []uint64
  RoleIDs   []uint64
}
```

Business code retrieves it via `permission.FromContext(ctx)`.

### Multi-Login Control

| Config | Description |
|--------|-------------|
| `multiLoginEnabled` | Allow simultaneous multi-device login |
| `maxLoginClient` | Maximum concurrent login devices |
| `revokeSessionOnHeartbeatOffline` | Revoke session when heartbeat times out |

When the limit is exceeded, the oldest session is evicted.

## LoginCrypto

Password security transport via RSA-OAEP challenge. Frontend must call `GetLoginCrypto` before submitting passwords.

```toml
[component.logincrypto]
challengeTTL = "3m0s"
timestampSkew = "2m0s"
rsaKeyBits = 4096
enableMetrics = true
```

### Flow Details

```text
Frontend                                  Backend
  |                                        |
  |-- GetLoginCrypto(username, ua) ------> |
  |                                        |-- Generate RSA key pair
  |                                        |-- Generate challengeId + nonce
  |                                        |-- Store in Redis, TTL = challengeTTL
  | <-------- publicKey, keyId,            |
  |           challengeId, nonce           |
  |                                        |
  |-- Web Crypto encrypt password -------> |
  |   RSA-OAEP + SHA-256                   |
  |                                        |
  |-- Login(passwordCipher, keyId,        |
  |    challengeId, nonce) ------------->  |
  |                                        |-- Verify challenge exists and not expired
  |                                        |-- Verify timestamp within timestampSkew
  |                                        |-- Decrypt password with private key
  |                                        |-- Verify password hash
  | <-------- accessToken, refreshToken    |
```

::: warning
Do not set `challengeTTL` too long to avoid replay attacks. 1-3 minutes is recommended for production.
:::

## IDGen

Snowflake ID generation, segment allocation, machine leases, namespace isolation, and stable ID codec.

```toml
[component.idgen.default]
namespace = "egoadmin-local"
name = "default"

[component.idgen.machine]
group = "egoadmin-local"

[component.idgen.codec]
secret = "local-stable-idcodec-secret"
```

### IDGen Usage

```go
type UserService struct {
  idGen *idgen.IDGenerator
}

func (s *UserService) CreateUser(ctx context.Context, cmd CreateUserCommand) error {
  userID := s.idGen.NextID()
  // userID is a globally unique snowflake ID
  user := &User{ID: userID, Username: cmd.Username}
  return s.repo.Create(ctx, user)
}
```

### IDCodec

Encodes numeric IDs into stable short strings for external API exposure:

```go
// Encode: numeric ID -> short string
encoded := idCodec.Encode(123456789) // e.g. "Kx7fM2"

// Decode: short string -> numeric ID
decoded, err := idCodec.Decode("Kx7fM2") // returns 123456789
```

::: warning
`idcodec` is reversible public ID encoding. It is not an authorization mechanism. Do not use encoded IDs in place of permission checks.
:::

## Redis / JetCache

### ERedis Modes

ERedis supports multiple Redis deployment modes:

| Mode | `mode` value | Use Case |
|------|-------------|----------|
| Standalone | `stub` | Local dev, single-node |
| Master-Slave | `master-slave` | Read-write separation |
| Sentinel | `sentinel` | High-availability auto failover |
| Cluster | `cluster` | Large-scale horizontal scaling |

Redis config:

```toml
[client.redis]
addr = "127.0.0.1:6380"
debug = true
mode = "stub"
password = "egoadmin"
db = 0
poolSize = 10
minIdleConns = 5
```

Usage example:

```go
type UserRepository struct {
  redis *eredis.ERedis
}

func (r *UserRepository) CacheUser(ctx context.Context, user *User) error {
  key := fmt.Sprintf("user:%d", user.ID)
  return r.redis.Set(ctx, key, user, 10*time.Minute)
}

func (r *UserRepository) GetCachedUser(ctx context.Context, id uint64) (*User, error) {
  key := fmt.Sprintf("user:%d", id)
  var user User
  if err := r.redis.Get(ctx, key, &user); err != nil {
    return nil, err
  }
  return &user, nil
}
```

### JetCache

JetCache is a multi-level cache framework supporting local cache + Redis remote cache:

```toml
[client.jetcache]
name = "default"
remoteExpiry = "1h0m0s"
localSize = 256
localExpiry = "1m0s"
refreshDuration = "2m0s"
stopRefreshAfter = "1h0m0s"
notFoundExpiry = "1m0s"
enableMetrics = true
enableSyncLocal = false
codec = "msgpack"
```

| Config | Description |
|--------|-------------|
| `remoteExpiry` | Redis remote cache TTL |
| `localSize` | Maximum local cache entries |
| `localExpiry` | Local cache TTL |
| `refreshDuration` | Background refresh interval |
| `stopRefreshAfter` | Stop refreshing after this duration without access |
| `notFoundExpiry` | Cache-miss protection TTL (prevents cache penetration) |
| `codec` | Serialization: `msgpack` or `json` |

Use cases: permission snapshots, user profiles, department trees, high-frequency read data.

```go
type PermissionCache struct {
  cache *jetcache.Cache
}

func (c *PermissionCache) GetUserMenus(ctx context.Context, userID uint64) ([]*Menu, error) {
  key := fmt.Sprintf("user_menus:%d", userID)
  var menus []*Menu
  if err := c.cache.Get(ctx, key, &menus); err != nil {
    return nil, err
  }
  return menus, nil
}
```

## AsyncQ

Async task queue built on Redis Lists with priority queues and retry support:

```toml
[client.asyncq]
redisAddr = "127.0.0.1:6380"
redisPassword = "egoadmin"
redisDB = 0
enableClient = true
enableServer = false
concurrency = 10
queues = { critical = 6, default = 3, low = 1 }
maxRetry = 3
retryDelayFunc = "exponential"
taskTimeout = "30s"
```

| Config | Description |
|--------|-------------|
| `enableClient` | Enable task producer |
| `enableServer` | Enable task consumer |
| `concurrency` | Concurrent workers per queue |
| `queues` | Queue names with priority weights |
| `maxRetry` | Maximum retry attempts |
| `retryDelayFunc` | Retry delay strategy (`exponential` = exponential backoff) |
| `taskTimeout` | Single task execution timeout |

### Task Producer and Consumer

```go
// Produce tasks
func (n *AuditNotifier) NotifyAudit(ctx context.Context, event *AuditEvent) error {
  return n.asyncQ.Enqueue(ctx, &asyncq.Task{
    Queue:   "default",
    Type:    "audit.notify",
    Payload: event,
  })
}

// Consume tasks
type AuditHandler struct{}

func (h *AuditHandler) Handle(ctx context.Context, task *asyncq.Task) error {
  event := task.Payload.(*AuditEvent)
  // Send notification, write audit log, etc.
  return nil
}
```

Use cases: audit log extensions, notifications, low-priority background processing, retryable tasks.

::: tip
In local development, set `enableServer = false` to only produce without consuming. In production, at least one service instance must have `enableServer = true`.
:::

## Component Rules

- Components provide reusable infrastructure, not service-specific business rules.
- Keep config minimal.
- Register resource cleanup with `shutdown.Manager`.
- Depend on narrow interfaces in business code.
