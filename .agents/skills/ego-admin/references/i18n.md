# Internationalization

Read this before adding or changing user-facing text in Go services, gateway upload/static handling, frontend pages/components, route/menu labels, validation messages, or error messages.

## Evidence Paths

- `internal/platform/i18n/message.go`
- `internal/platform/i18n/locales/active.zh-CN.toml`
- `internal/platform/i18n/locales/active.en.toml`
- `internal/client/userclient/client.go`
- `web/src/i18n/index.ts`
- `web/src/i18n/package/zh-CN.ts`
- `web/src/i18n/package/en-US.ts`
- `web/src/components/LanguageSwitcher/index.vue`
- `web/src/api/index.ts`
- `web/src/App.vue`
- `web/src/config/routeMenu.ts`

## Backend Pattern

Backend i18n uses `github.com/nicksnyder/go-i18n/v2/i18n` with embedded TOML files. Default language is `zh-CN`; supported defaults are `zh-CN` and English.

Rules:

- Put backend message IDs in both `active.zh-CN.toml` and `active.en.toml`.
- Use stable PascalCase message IDs, for example `DataPermissionOutOfScope`.
- Use `platformi18n.Message(ctx, id)` or `platformi18n.Localize(ctx, id, data)` for plain text.
- Use typed wrappers for business errors:
  - `platformi18n.ErrorFailed(ctx, id, data)`
  - `platformi18n.ErrorNotLogin(ctx, id, data)`
  - `platformi18n.ErrorAccessDenied(ctx, id, data)`
- Do not return raw user-facing strings from service/controller code.
- Keep technical details in logs, not in localized user messages.

Example:

```go
return platformi18n.ErrorAccessDenied(ctx, "DataPermissionOutOfScope", nil)
```

Parameterized TOML messages use go-i18n template data:

```toml
[RoleInUseCount]
other = "该角色已被 {{.Count}} 个用户使用"
```

```go
return platformi18n.ErrorFailed(ctx, "RoleInUseCount", map[string]any{"Count": count})
```

## Language Propagation

Frontend requests send `Accept-Language` based on `web/src/i18n/index.ts`. Gateway/user gRPC clients must forward `accept-language` metadata so downstream services localize errors and messages correctly. Do not drop this metadata in `internal/client/*` wrappers.

Gateway must preserve remote EGO errors by converting gRPC status details back to `*eerrors.EgoError`; otherwise localized access/auth/business errors can be remapped to `内部错误`.

## Frontend Pattern

Frontend i18n uses `vue-i18n` with:

- `web/src/i18n/index.ts` for setup, locale normalization, persistence, and `getAcceptLanguage`.
- `web/src/i18n/package/zh-CN.ts` and `en-US.ts` for language packages.
- `LanguageSwitcher` for changing language.
- Element Plus global locale should stay wired through the application root when present.

Rules:

- Add every key to both frontend language packages.
- Prefer nested domain keys such as `role.addSuccess`, `menu.roleManagement`, and `error.loginExpired`.
- Use `t('path.to.key')` in templates and scripts; do not hardcode Chinese/English UI strings in new code.
- Use interpolation for variables, for example `{count}`.
- Route `meta.title` and `routeMenu.title` should be i18n keys.
- Keep keyword/product names untranslated when they are identifiers, API names, usernames, role names, or user-entered content.

## API And Error Display

- API modules stay under `web/src/api/**`; do not localize protobuf field names.
- HTTP/gRPC error payloads should use backend-localized `message` with stable `reason`.
- Frontend login-state handling should use stable backend reasons where possible and local fallback text from `error.*`.

## Adding A New User-Facing Feature

1. Add backend message IDs for all service/controller business errors.
2. Add frontend keys for labels, placeholders, buttons, dialogs, empty states, and success/error toasts.
3. Replace literal strings in routes, routeMenu, page components, and helpers with keys.
4. Ensure `Accept-Language` is sent and forwarded across gateway to user service.
5. Add tests for backend localization or gateway-visible error preservation when behavior is sensitive.

## Validation

- `go test -race ./internal/platform/i18n ./internal/client/userclient`
- Relevant backend service tests when errors changed.
- `cd web && pnpm run type-check`
- `cd web && pnpm run build`
- `make e2e E2E_TIMEOUT=20m` when localized behavior is gateway-visible.
