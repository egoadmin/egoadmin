为什么不一样

三个让它与众不同的设计前提

不是又一个后台脚手架。它把 AI 协作、接口契约、权限模型当作第一性原则来设计。

Principle 01

Agent First

框架自带一套结构化 AI skill 文档，覆盖 proto 契约、权限、分布式事务、数据权限、迁移、停机等每一层。AI 不靠猜你的架构，而是按既定约定读对参考、改对链路。

解决：AI 在陌生框架里乱写、跨层污染、破坏约定。你和 AI 共享同一套事实来源。
Principle 02

Proto First

.proto 是唯一事实来源。make gen 一条命令，从 proto 生成 gRPC + HTTP 双栈、参数校验、OpenAPI 文档、API catalog、业务错误码、响应映射与前端类型。

解决：接口、文档、校验、前端类型各写一遍还对不上。这里全部从 proto 推导，天然一致。
Principle 03

契约化权限

权限契约（permission-contract）由代码自动生成，不需要人工维护菜单表、权限码、按钮配置。路由即菜单，API 标识即权限点；前端勾选必须落在契约内，后端按契约强制校验边界，契约可 diff、可审计。

解决：传统后台手工配菜单树 + 权限码，易错、难审计、易越权。这里一切基于契约，还内建防越权授权。
不止于理念

生产级能力，开箱即用

这些通常要你自己造的轮子，框架已经接好并经过 e2e 验证。

真·微服务底座

  • 三服务拆分：gateway / user / idgen 独立进程、独立配置，gRPC + etcd 服务发现，跨服务调用强制走 client 封装
  • DTM 分布式事务：Saga / TCC / 事务消息预接好，跨服务写一致性 + 幂等屏障表
  • 优雅停机：readiness 摘流 + 反序关闭资源，K8s 滚动更新零丢请求
  • 按库迁移：每服务独立 Atlas 版本化迁移，可审计的 schema 演进，拒绝生产 AutoMigrate

省掉重复造的轮子

  • 加密登录：RSA-OAEP 密文传输，无明文/MD5 密码，挑战一次性防重放；真会话生命周期（强制下线 / 吊销）
  • 行级数据权限：DataScope 组织树可见性（全部 / 本部门及子级 / 仅本部门 / 仅本人）+ 防越权授权
  • 分布式 ID：idgen 号段分配 + idcodec 可逆公开 ID，不暴露自增主键
  • 文件与前端：TUS 断点续传 + MinIO + 签名 CDN 图片处理；单二进制内嵌 SPA；端到端 i18n 错误消息跨服务不丢
